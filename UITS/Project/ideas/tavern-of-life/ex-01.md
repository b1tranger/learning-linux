To understand how your game works, think of the terminal as a grid of cells (rows and columns). tput allows us to pick exactly which cell to type in.
Here is the breakdown of the two most important parts of your script:
## 1. The draw_char() Logic
This function is your Renderer. Its only job is to look at the current x and y coordinates and draw the character there.

draw_char() {
    # 1. Move cursor to the current (y, x) position
    tput cup $y $x 
    echo " o " # The Head

    # 2. Draw the body on the row below (y+1)
    tput cup $((y+1)) $x
    echo "/|\\" 

    # 3. Draw the legs on the row below that (y+2)
    # The 'frame' variable toggles every move to create the animation
    tput cup $((y+2)) $x
    if [ $((frame % 2)) -eq 0 ]; then
        echo "/ \\" # Frame A: Wide legs
    else
        echo " | "  # Frame B: Closed legs (walking)
    fi
}


* The Animation Trick: We use the modulo operator % 2. Since frame increases by 1 every time you move, the result alternates between 0 and 1, switching the leg style.

## 2. The "Move Logic" (Inside the while loop)
Movement in a terminal game isn't "sliding"; it is actually Delete -> Change Math -> Redraw.
## Step A: The Footprint Clear
Before we change the x or y values, we have to "erase" where the character currently is. If we don't, as the character moves, it will leave a trail of old body parts behind.

for i in {0..2}; do 
    tput cup $((y+i)) $x
    echo "   " # Overwrites the character with 3 blank spacesdone

## Step B: The Math Update
When you press a key, we change the coordinate values. We also add Collision Detection (the if checks) to make sure you don't go through the walls.

case $input in
    w) # UP
        # Only decrease Y if we are not at the top border (row 1)
        [[ $y -gt 1 ]] && ((y--)) 
        ;;
    d) # RIGHT
        # Only increase X if we are not at the right border (WIDTH minus character width)
        [[ $x -lt $((WIDTH-3)) ]] && ((x++)) 
        ;;esac

## Step C: The Redraw
Now that the coordinates are updated, the loop starts over, calls draw_char(), and the character appears in the new spot. Because this happens in milliseconds, it looks like smooth movement to the human eye.
## Summary of the Cycle:

   1. Input: Player presses d.
   2. Erase: Put spaces at old x,y.
   3. Calculate: x becomes x + 1.
   4. Draw: Put character at new x,y.
