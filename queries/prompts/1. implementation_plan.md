# Modify Tavern of Life Game

This plan covers adding a character legend on the right side of the window, implementing a sequential puzzle logic to obtain keys, and adding a poetry reward from the Old Man.

## Proposed Changes

### 1. Refactor Character Draw Functions
We will modify the existing `draw_bartender`, `draw_patron`, `draw_drunk`, and `draw_oldman` functions to accept optional `$x` and `$y` parameters. This allows us to reuse the exact same drawing logic and animations for a "legend" on the right side of the screen without writing new drawing code.
- If parameters are provided, the function will draw at those coordinates.
- If not provided, it will default to the original map coordinates (e.g., `BARTENDER_X`).

### 2. Add Character Legend (Roles)
We will add a new `draw_legend()` function that uses the refactored draw functions to draw each character on the right side of the game window (starting around X=65).
- Beside each drawn character, we will print their role (e.g., "Bartender", "Sober Patron", "Drunk Patron", "Old Man").
- This function will be called inside `draw_all()` so the legend characters animate in sync with the map characters.

### 3. Implement Sequential Key Puzzle
We will introduce a `KEY_STAGE` variable to track the player's progress in obtaining keys.
The sequence of obtaining keys will be:
1. **Sober Patron** (Key 1)
2. **Drunk Patron** (Key 2)
3. **Old Man** (Key 3)
4. **Bartender** (Final Key, unlocks the exit)

Changes to the conversation logic (`check_talk()`):
- **Sober Patron**: Gives Key 1 after finishing his QnA.
- **Drunk Patron**: If the player hasn't obtained Key 1, he will tell the player to talk to the Sober Patron first. If Key 1 is obtained, finishes QnA and gives Key 2.
- **Old Man**: If the player hasn't obtained Key 2, he will tell the player to seek wisdom from the Drunk Patron. If Key 2 is obtained, finishes QnA and gives Key 3.
- **Bartender**: If the player hasn't obtained Key 3, he will tell the player to speak with the others first. If Key 3 is obtained, finishes QnA and gives the Final Key (`has_realization=true`), unlocking the exit.

### 4. Old Man's Poem
Once the player has the Final Key (`has_realization=true`), talking to the Old Man again will trigger a new dialogue where he shares a short poem as a reward for clearing the game before the player exits.
